    Support & Access¶

    This page provides contact information, troubleshooting resources, and self-service solutions for Signal Pilot Suite users. For most issues, the self-service options below may provide faster resolution than waiting for an email response.

    Contact Information¶

    • Website: https://www.signalpilot.io
    • Email: support@signalpilot.io
    • Documentation: https://docs.signalpilot.io
    • GitHub Issues: Report technical issues

    Getting Access¶

    After purchasing the Signal Pilot Suite:

    1. Provide Your TradingView Username: Send your exact TradingView username to support@signalpilot.io
    2. Wait for Invite: Activation is typically completed within 24 hours (often much faster during business hours)
    3. Check Your Indicators List: Once invited, the indicators will appear in your TradingView "Invite-only scripts" section
    4. Add to Chart: Search for "Signal Pilot" in your indicators list and add them to your chart

    Note: Access is tied to your TradingView username. If you change your username, contact support to update your access.

    Before Contacting Support¶

    Please try these self-service steps first. Most issues can be resolved quickly without waiting for an email response:

    Pre-Contact Checklist¶

    • Check the FAQ page for answers to common questions
    • Review the Troubleshooting Guide for technical issues
    • Consult the specific indicator documentation page for configuration help
    • Search this documentation site using the search bar (top of page)
    • Clear your browser cache and refresh TradingView
    • Try the indicator on a different symbol/timeframe to isolate the issue
    • Check if your TradingView plan supports the features you're trying to use (some features require Premium/Pro+)

    Troubleshooting Common Issues¶

    Installation & Access Issues¶

    Issue: Cannot find Signal Pilot indicators

    Possible Causes & Solutions:

    • Not yet invited: Check your email for confirmation of access. Invites typically process within 24 hours.
    • Wrong TradingView account: Ensure you're logged into the TradingView account you provided for access.
    • Looking in wrong place: Indicators are in the "Invite-only scripts" section, not public indicators. Search for "Signal Pilot" in your indicators menu.
    • Username mismatch: Verify you provided the correct TradingView username (not email). Contact support to update if needed.

    Issue: Indicators not loading on chart

    Possible Causes & Solutions:

    • TradingView plan limitations: Free plans are limited to 3 indicators per chart. Upgrade to Premium for more indicators.
    • Browser issues: Clear browser cache, disable browser extensions, try incognito/private mode.
    • Outdated browser: Update to the latest version of Chrome, Firefox, Safari, or Edge.
    • Script error: Check the TradingView console (F12 ‚Üí Console tab) for error messages and report to support.

    Issue: Indicator shows "Script could not be loaded"

    Possible Causes & Solutions:

    • Access revoked/expired: Contact support to verify your subscription status.
    • TradingView service issues: Check TradingView Status for platform outages.
    • Network/firewall: Disable VPN, check firewall settings, try different network connection.

    Alert Setup Problems¶

    Issue: Alerts not firing

    Possible Causes & Solutions:

    • Alert limit reached: TradingView has alert limits based on plan. Check Settings ‚Üí Alerts to see active alerts.
    • Incorrect condition: Review alert configuration. Ensure you've selected the correct indicator output.
    • Expired alert: Alerts expire if you close the chart. Keep chart open or set "Once Per Bar Close" frequency.
    • Wrong timeframe: Alert was created on one timeframe but you're viewing another. Match alert timeframe to chart.
    • Indicator removed: If you remove an indicator, its alerts stop working. Re-add indicator and recreate alerts.

    Issue: Too many alerts (alert spam)

    Possible Causes & Solutions:

    • Wrong frequency setting: Change alert frequency to "Once Per Bar Close" instead of "Once Per Bar".
    • Too sensitive settings: Adjust indicator sensitivity settings (thresholds, filters, etc.).
    • Multiple identical alerts: Check for duplicate alerts on the same condition. Delete extras.
    • Lower timeframe: Consider using higher timeframes (e.g., H1 instead of M5) for fewer, higher-quality signals.

    Issue: Alert webhook not working

    Possible Causes & Solutions:

    • Invalid URL: Verify webhook URL is correct and includes https://.
    • Webhook service down: Test webhook endpoint using a tool like Postman or RequestBin.
    • JSON formatting error: Validate JSON syntax in alert message. Use a JSON validator tool.
    • TradingView Premium required: Webhooks require TradingView Premium or higher plan.
    • Rate limiting: Some webhook services rate-limit requests. Check service logs.

    See the Webhooks Guide for detailed webhook setup instructions.

    Performance & Display Issues¶

    Issue: Indicator loads slowly or chart freezes

    Possible Causes & Solutions:

    • Too many indicators: Reduce number of active indicators. Try using Omnideck (all-in-one) instead of multiple separate indicators.
    • Too much historical data: Reduce chart range. Load 6 months or less for optimal performance.
    • Low-end device: Close other browser tabs, use TradingView desktop app, or upgrade device.
    • Complex settings: Some settings combinations are resource-intensive. Try default settings first.

    Issue: Visual elements not displaying correctly

    Possible Causes & Solutions:

    • Hidden by other indicators: Adjust z-order or disable overlapping indicators temporarily.
    • Color settings: Check indicator color settings. Try adjusting brightness/opacity.
    • Chart theme conflict: Switch between light/dark chart themes to test.
    • Display settings disabled: Review indicator settings to ensure visual components are enabled (lines, labels, boxes, etc.).

    Issue: Signals different from screenshots/documentation

    Possible Causes & Solutions:

    • Different settings: Documentation uses default settings. Compare your settings to defaults.
    • Different timeframe/symbol: Signals vary by market conditions. Try documented examples (symbol + timeframe).
    • Repainting concern: Signal Pilot indicators are non-repainting. Apparent changes are due to new data, not repainting. See Non-Repainting Policy.
    • Version mismatch: Ensure you're using the latest version (v1.0). Old versions may behave differently.

    Common Error Messages¶

    Error Message Meaning Solution
    "Study Error" Script execution failed Refresh chart, check symbol data availability, try different symbol
    "Too many requests" Rate limit hit Wait 60 seconds, reduce number of active indicators
    "Cannot access this symbol" Data not available Indicator may not support this data type (crypto futures, etc.). Try different symbol
    "Script has too many calculations" Exceeds TradingView limits Reduce chart historical range, disable complex features
    "No data" Insufficient historical data Symbol too new or illiquid. Try well-established symbols (BTC, ES, EURUSD)

    For additional error messages and solutions, see the comprehensive Troubleshooting Guide.

    Contacting Support¶

    If self-service options don't resolve your issue, please contact support with the following information for faster resolution:

    Support Priority Levels¶

    High Priority (24-hour response goal):

    • Cannot access indicators after purchase confirmation
    • Indicators causing TradingView crashes or severe performance issues
    • Billing/payment issues
    • Security concerns

    Standard Priority (2-3 business days):

    • Configuration help and "how-to" questions
    • Alert setup assistance
    • Feature requests and suggestions
    • Documentation clarifications
    • Potential bug reports (non-critical)

    Note: Response times are goals, not guarantees. Actual response time may vary based on support volume and issue complexity.

    How to Report an Issue¶

    When contacting support, please include:

    1. Your TradingView Username: Helps verify access and subscription status
    2. Indicator Name & Version: e.g., "Pentarch v1.0" or "Omnideck v1.0"
    3. Symbol & Timeframe: e.g., "BTCUSDT on 15-minute chart"
    4. Issue Description: Clear, detailed explanation of what's happening vs. what you expected
    5. Steps to Reproduce: What actions lead to the issue?
    6. Screenshots: Visual examples are extremely helpful (use imgur.com or similar if emailing)
    7. Settings Used: Screenshot of indicator settings panel if using custom configuration
    8. Browser & Device: e.g., "Chrome 120 on Windows 11" or "Safari on MacBook Pro"
    9. TradingView Plan: Free, Essential, Plus, Premium, or Premium+
